# 数据库配置

本文档详细介绍 NexusAuth 数据库的配置过程。

## 使用 Docker 运行 PostgreSQL

### 启动数据库

```bash
docker run -d \
  --name nexusauth-db \
  -e POSTGRES_PASSWORD=your-secure-password \
  -e POSTGRES_USER=nexusauth \
  -e POSTGRES_DB=nexusauth \
  -p 5432:5432 \
  postgres:16
```

### 验证数据库

```bash
docker exec -it nexusauth-db psql -U nexusauth -d nexusauth -c "SELECT version();"
```

## 使用现有 PostgreSQL

### 创建数据库

```sql
CREATE USER nexusauth WITH PASSWORD 'your-secure-password';
CREATE DATABASE nexusauth OWNER nexusauth;
GRANT ALL PRIVILEGES ON DATABASE nexusauth TO nexusauth;
```

## 配置连接字符串

### NexusAuth.Host

编辑 `src/NexusAuth.Host/appsettings.json`：

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Host=localhost;Port=5432;Database=nexusauth;Username=nexusauth;Password=your-secure-password"
  }
}
```

## 运行迁移

### 生成迁移

```bash
cd src/NexusAuth.Host
dotnet ef migrations add InitialCreate
```

### 执行迁移

```bash
dotnet ef database update
```

## 数据库表说明

### 核心表

| 表名 | 说明 |
|------|------|
| users | 用户表 |
| oauth_clients | OAuth 客户端 |
| api_resources | API 资源（Scope） |
| authorization_codes | 授权码 |
| refresh_tokens | 刷新令牌 |

### oauth_clients 表结构

```sql
CREATE TABLE oauth_clients (
    id UUID PRIMARY KEY,
    client_id VARCHAR(256) UNIQUE NOT NULL,
    client_secrets JSONB,
    token_endpoint_auth_method VARCHAR(64),
    client_name VARCHAR(256),
    description TEXT,
    redirect_uris JSONB,
    post_logout_redirect_uris JSONB,
    allowed_scopes JSONB,
    allowed_grant_types JSONB,
    require_pkce BOOLEAN DEFAULT true,
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

### api_resources 表结构

```sql
CREATE TABLE api_resources (
    id UUID PRIMARY KEY,
    name VARCHAR(256) UNIQUE NOT NULL,
    display_name VARCHAR(256),
    audience VARCHAR(256),
    description TEXT,
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

## 常见问题

### 端口被占用

```bash
# 查看端口占用
lsof -i :5432

# 或使用 netstat
netstat -an | grep 5432
```

### 连接失败

检查 `pg_hba.conf` 是否允许连接：

```bash
# 在容器内检查
docker exec -it nexusauth-db cat /etc/postgresql/16/main/pg_hba.conf
```
